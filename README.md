# KitabiAI — Intelligent Arabic Book Platform

KitabiAI is a full-stack platform that transforms PDF books — primarily Arabic — into structured, searchable, and conversational knowledge. Upload a PDF and the system automatically extracts the table of contents, splits the book into sections, generates AI summaries, fills in metadata, and creates a public page where anyone can read and chat with the book.

**Live demo**: [kitabiai.azurewebsites.net/library](https://kitabiai.azurewebsites.net/library)

---

## Features

### Book Processing Pipeline
- **Dual TOC extraction modes**: extract from an existing table of contents page, or auto-generate by detecting headings with Azure Document Intelligence
- **Content quality filters**: strips headers, footers, page numbers, footnotes, and repeated section titles using Azure paragraph role metadata and Y-position margin detection
- **Two-line title merging**: detects headings split across two lines and merges them correctly
- **Skip pages control**: configurable page offset to skip cover, license, and front matter pages
- **Multi-format output**: HTML (RTL-aware), Markdown, JSONL — uploaded to Azure Blob Storage

### AI Summarization
- Section-level summaries with structured format (overview + key highlights)
- Full-book summary generated from section summaries
- Auto-fill metadata from book summary: description, category (from a predefined clean list), and keywords — only fills empty fields, never overwrites admin data
- Page-range fallback for books without stored content

### RAG Chatbot (per-book)
- Each book gets a public page at `/books/{id}` with a chat widget
- Chunk-level embeddings: sections are split into ~400-word overlapping chunks, each embedded with `text-embedding-3-small`
- Vector similarity search via pgvector (`<=>` cosine distance)
- Answers generated by GPT-4o-mini with source citations (section title + page range)
- Full Arabic RTL support throughout

### Library & Admin
- Public library with real-time filtering by language, author, category, and keyword search
- Dark mode, responsive layout, book cover images
- Admin panel: upload, edit, hide/show books, summarize, embed, view public page
- Accurate page and section statistics (visible vs. all)

## Setup

### Prerequisites
- Python 3.11
- PostgreSQL with pgvector extension enabled
- Azure Document Intelligence API key
- OpenAI API key
- DeepSeek API key

### Local Development

```bash
# Clone and install
git clone https://github.com/suhaJamal/KitabiAI.git
cd KitabiAI
python -m venv venv
venv\Scripts\activate       # Windows
pip install -r requirements.txt

# Configure environment
cp .env.example .env
# Fill in DATABASE_URL, AZURE_*, OPENAI_API_KEY, DEEPSEEK_API_KEY

# Run
uvicorn app.main:app --reload --port 8000
```

### Database Migration (pgvector)

Run once in your PostgreSQL client after enabling the vector extension:

```sql
CREATE EXTENSION IF NOT EXISTS vector;

ALTER TABLE sections
  ADD COLUMN IF NOT EXISTS summary    TEXT,
  ADD COLUMN IF NOT EXISTS embedding  vector(1536);

CREATE TABLE IF NOT EXISTS section_chunks (
    id          SERIAL PRIMARY KEY,
    section_id  INTEGER NOT NULL REFERENCES sections(id)  ON DELETE CASCADE,
    book_id     INTEGER NOT NULL REFERENCES books(id)     ON DELETE CASCADE,
    chunk_index INTEGER NOT NULL,
    content     TEXT    NOT NULL,
    embedding   vector(1536)
);

CREATE INDEX IF NOT EXISTS section_chunks_book_idx
    ON section_chunks (book_id);
```

### Required Environment Variables

```
DATABASE_URL=postgresql://user:pass@host/db
```

## Key Workflows

### Upload a book
1. Go to `/` → fill in metadata, choose TOC method (extract or generate), upload PDF
2. System runs Azure Document Intelligence, extracts TOC, splits sections, generates HTML/Markdown/JSONL, stores in Azure Blob Storage

### Summarize a book
1. Admin panel → expand book row → **Summarize**
2. System generates section summaries, then book summary, then auto-fills empty metadata fields

### Enable chat for a book
1. Admin panel → expand book row → **Embed**
2. System splits each section into ~400-word chunks and embeds each with OpenAI
3. Public page at `/books/{id}` now shows the chat widget
